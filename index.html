          Avant de commencer

          Loyer Manager — sur votre serveur, sans cloud.

          Loyer Manager vous aide à suivre les loyers, enregistrer les virements et produire des quittances. L'application tourne sur votre serveur : aucun compte cloud, vos données restent sur le disque.

          Accès : ouvrez l'application via son URL (http://…/) — pas en double-cliquant sur le fichier.
          Import rapide : glissez-déposez un fichier .json (données) ou .csv (relevé bancaire) n'importe où sur la page.

          Organisation des fichiers (serveur)

          LoyerManager/
├── index.html
├── api.php
├── data/
│   └── loyer-data.json     ← paramètres, virements, registre modèles
└── templates/
    ├── quittances/
    │   └── principal.html
    └── mails/
        ├── principal.html
        └── principal-subject.txt

          Les fichiers *.example.* servent de référence. L'application crée vos fichiers personnalisés via api.php si besoin.

          Étape 1 Premiers pas

          Ouvrez l'application via l'URL de votre serveur. Au premier lancement, le badge affiche Enregistré en vert : vos données sont lues et écrites automatiquement via api.php.

          1. Vérifiez que nginx et PHP fonctionnent (curl http://localhost/api.php?action=status).
          2. Ouvrez Paramètres et renseignez bailleur, locataire et montant du loyer.
          3. Cliquez sur le bouton bleu Enregistrer en bas à droite.

          Étape 2 Paramétrer l'application

          Rendez-vous dans l'onglet Paramètres et remplissez les sections dans l'ordre ci-dessous. Cliquez sur le bouton bleu Enregistrer en bas à droite quand vous avez terminé.

          2.1 — Location

          • Début de location : date de prise d'effet du bail.
          • Jour théorique du virement : jour habituel du paiement (souvent le 1er du mois).

          2.2 — Bailleur et locataire

          Renseignez les noms et adresses : ils apparaîtront sur la quittance de loyer.

          2.3 — Signature

          Importez une photo ou un scan de votre signature (PNG/JPG, max. 5 Mo). Elle sera placée en bas de la quittance.

          2.4 — Historique des loyers

          Indiquez le montant actuel du loyer et la date à partir de laquelle il s'applique. Si le loyer a changé dans le passé, ajoutez un palier pour chaque période.

          2.5 — Émetteurs (pour l'import bancaire)

          Si vous importez un relevé CSV, configurez le nom du locataire et les mots à repérer dans le libellé bancaire (ex. Jean Dupont).

          2.6 — Modèles quittance et mail

          Dans Paramètres → Modèles, gérez vos modèles. Le modèle principal (principal) est toujours présent, en lecture seule : aperçu et export uniquement. Pour le personnaliser, utilisez + Nouveau modèle ou Importer modèle (crée un nouveau modèle, ne remplace jamais le principal). L'édition du contenu se fait dans les onglets Quittance et Mail (mode « Édition du modèle »). Le panneau Mots-clés propose les mêmes variables pour quittance et mail ({{paiement}}, {{bailleur.name}}, etc.).

          2.7 — Destinataires mail (optionnel)

          Indiquez les adresses e-mail (À, CC, CCI) et la signature texte ({{signature}}).

          Étape 3 Ajouter les virements

          Deux méthodes sont possibles — choisissez celle qui vous convient.

          Méthode A — Import CSV (relevé bancaire)

          1. Connectez-vous à votre banque en ligne et exportez vos opérations au format CSV (relevé de compte).
          2. Dans l'onglet Virements, cliquez sur Importer CSV et choisissez le fichier téléchargé.
          3. Une liste s'affiche : cochez les virements à enregistrer (les doublons sont déjà exclus).
          4. Cliquez sur Importer la sélection.
          L'import repère automatiquement les virements du locataire grâce aux « émetteurs » configurés à l'étape 2.5.

          Méthode B — Saisie manuelle

          1. Cliquez sur + Ajouter un virement.
          2. Indiquez la date, le montant et l'émetteur.
          3. Cliquez sur Enregistrer.

          Utilisez la saisie manuelle pour corriger une ligne ou ajouter un virement ponctuel sans fichier bancaire.

          Étape 4 Consulter les données

          L'onglet Tableau de bord centralise la vue d'ensemble.

          • Utilisez la barre de période sous le menu : sélecteur de mois, timeline colorée, ou « Afficher une période » pour une plage.
          • Les indicateurs résument le solde à date, le recouvrement et les mois problématiques.
          • Le tableau « Détail par mois » affiche un statut (payé, partiel, impayé…) et un éventuel retard (+X j).
          • La heatmap colore chaque mois — cliquez une cellule pour la sélectionner.
          • Les graphiques et le récapitulatif annuel reprennent les mêmes chiffres sous forme visuelle.
          • Imprimer le rapport pour une vue papier (tableaux et heatmap).

          Étape 5 Quittance et export

          1. Choisissez le mois dans l'onglet Quittance (ou Tableau de bord / Mail — même sélecteur).
          2. L'aperçu se remplit automatiquement à partir du modèle choisi.
          3. Pour modifier le modèle : basculez en Édition du modèle (modèles personnalisés uniquement — le principal est en lecture seule).
          4. Exportez :
            • PDF — le plus courant pour envoyer ou imprimer ;
            • DOCX — pour modifier dans Word ;
            • HTML — page web autonome ;
            • Exporter plusieurs… — un seul fichier PDF, DOCX ou HTML pour une plage de mois.
          Cliquez sur Régénérer après avoir modifié des virements ou des paramètres pour recalculer l'aperçu.

          Étape 6 Mail et envoi

          1. Choisissez le mois dans l'onglet Mail (sélecteur partagé avec Tableau de bord et Quittance).
          2. Objet et corps s'affichent pour le modèle sélectionné.
          3. Pour personnaliser le modèle : Nouveau modèle… ou Importer modèle (.json), puis mode Édition du modèle.
          4. Utilisez le panneau Mots-clés (identique à la quittance) pour insérer {{paiement}}, adresses, etc.
          5. Envoyez via EML + PDF (fichier .eml avec quittance jointe) ou mailto (attachez le PDF manuellement).
          Les destinataires (À, CC, CCI) se configurent dans Paramètres → Destinataires et signature mail.

          Installation sur un serveur (recommandé)

          Hébergez l'application sur un serveur web avec PHP. Vos données restent sur le disque du serveur (data/, templates/).

          Étapes

          1. Clonez ou uploadez le projet sur le serveur.
          2. Copiez config.example.php en config.php (optionnel : api_key en production).
          3. Vérifiez que PHP peut écrire dans data/ et templates/ (droits 755 ou 775 dans le panneau hébergeur).
          4. Ouvrez l'application via son URL (https://votre-domaine.fr/) — pas en double-cliquant sur le fichier.

          Le badge affiche Enregistré lorsque api.php répond correctement.

          Test en local

          Dans le dossier du projet : php -S localhost:8080 puis ouvrez http://localhost:8080/. Aucun script ni dossier deploy/ n'est nécessaire.

          Jeu de démonstration

          cp docs/demo/loyer-data.demo.json data/loyer-data.json — voir docs/demo/README.md.

          HTTPS et mot de passe

          Sur une URL publique, activez HTTPS et un mot de passe devant le site (outil « protection de répertoire » de votre hébergeur, ou configuration nginx/Apache). Complétez avec une clé API dans config.php. Guide complet : docs/SECURITE.md et docs/HEBERGEMENT-MUTUALISE.md (mutualisé sans terminal).

          Sécurité : trois couches recommandées — HTTPS, mot de passe site, clé API. Pas de comptes ni base de données. Exportez régulièrement une copie via Paramètres → Données.

          Dépannage

          Les modèles ne s'enregistrent pas

          Vérifiez nginx/php-fpm ou Apache, les droits d'écriture sur templates/, et que le badge affiche Enregistré. Test : curl https://votre-site/api.php?action=status.

          Impossible de modifier le modèle principal

          C'est normal : le modèle principal est en lecture seule. Dupliquez-le ou importez un fichier pour créer une variante personnalisée.

          Le badge indique « Serveur indisponible »

          Ouvrez l'app via http:// (pas file://). Vérifiez nginx, php8.4-fpm et les droits sur data/.

          Le fichier JSON est corrompu

          Au démarrage, l'application vous propose de télécharger une copie du fichier endommagé, puis de le recréer vide.

          L'import CSV ne trouve aucun virement

          Vérifiez les « Émetteurs » dans Paramètres : les mots-clés doivent correspondre au libellé affiché par votre banque.

          Les chiffres du tableau de bord semblent incorrects

          Vérifiez la date de début de location, l'historique des loyers et que tous les virements ont bien été saisis.

          Faire une copie de secours

          Paramètres → Données → Copie de secours (export) télécharge votre fichier JSON.

          Exporter plusieurs quittances

          Génère un fichier unique contenant une quittance par mois dans la plage choisie (modèle sélectionné dans l'onglet).
