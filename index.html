          Avant de commencer

          Loyer Manager — sur votre serveur, sans cloud.

          Loyer Manager vous aide à suivre les loyers, enregistrer les virements et produire des quittances. L'application tourne sur votre serveur : aucun compte cloud, vos données restent sur le disque.

          Accès : ouvrez l'application via son URL (http://…/index.html) — pas en double-cliquant sur le fichier.

          Organisation des fichiers (serveur)

          LoyerManager/
├── index.html
├── api.php
├── data/
│   └── loyer-data.json     ← paramètres, virements, registre modèles
└── templates/
    ├── quittances/
    │   └── principal.html
    └── mails/
        ├── principal.html
        └── principal-subject.txt

          Les fichiers *.example.* servent de référence. L'application crée vos fichiers personnalisés via api.php si besoin.

          Étape 1 Premiers pas

          Ouvrez l'application via l'URL de votre serveur. Au premier lancement, le badge affiche Enregistré en vert : vos données sont lues et écrites automatiquement via api.php.

          1. Vérifiez que nginx et PHP fonctionnent (curl http://localhost/api.php?action=status).
          2. Ouvrez Paramètres et renseignez bailleur, locataire et montant du loyer.
          3. Cliquez sur le bouton bleu Enregistrer en bas à droite.

          Étape 2 Paramétrer l'application

          Rendez-vous dans l'onglet Paramètres et remplissez les sections dans l'ordre ci-dessous. Cliquez sur le bouton bleu Enregistrer en bas à droite quand vous avez terminé.

          2.1 — Location

          • Début de location : date de prise d'effet du bail.
          • Jour théorique du virement : jour habituel du paiement (souvent le 1er du mois).

          2.2 — Bailleur et locataire

          Renseignez les noms et adresses : ils apparaîtront sur la quittance de loyer.

          2.3 — Signature

          Importez une photo ou un scan de votre signature (PNG/JPG, max. 400 Ko). Elle sera placée en bas de la quittance.

          2.4 — Historique des loyers

          Indiquez le montant actuel du loyer et la date à partir de laquelle il s'applique. Si le loyer a changé dans le passé, ajoutez un palier pour chaque période.

          2.5 — Émetteurs (pour l'import bancaire)

          Si vous importez un relevé CSV, configurez le nom du locataire et les mots à repérer dans le libellé bancaire (ex. MARYSE VALLEE).

          2.6 — Modèles quittance et mail

          Dans Paramètres → Modèles, personnalisez le HTML avec des mots-clés {{mois}}, {{bailleur.name}}, etc. Deux modes : Modifier le modèle (mots-clés visibles) et Aperçu (données réelles). Cliquez Enregistrer pour écrire dans templates/quittance.html, templates/mail.html et templates/mail-subject.txt. Réinitialiser remet le modèle par défaut embarqué.

          2.7 — Destinataires mail (optionnel)

          Indiquez les adresses e-mail des destinataires et la signature texte ({{signature}}).

          Étape 3 Ajouter les virements

          Deux méthodes sont possibles — choisissez celle qui vous convient.

          Méthode A — Import CSV (relevé bancaire)

          1. Connectez-vous à votre banque en ligne et exportez vos opérations au format CSV (relevé de compte).
          2. Dans l'onglet Virements, cliquez sur Importer CSV et choisissez le fichier téléchargé.
          3. Une liste s'affiche : cochez les virements à enregistrer (les doublons sont déjà exclus).
          4. Cliquez sur Importer la sélection.
          L'import repère automatiquement les virements du locataire grâce aux « émetteurs » configurés à l'étape 2.5.

          Méthode B — Saisie manuelle

          1. Cliquez sur + Ajouter un virement.
          2. Indiquez la date, le montant et l'émetteur.
          3. Cliquez sur Enregistrer.

          Utilisez la saisie manuelle pour corriger une ligne ou ajouter un virement ponctuel sans fichier bancaire.

          Étape 4 Consulter les données

          L'onglet Tableau de bord centralise la vue d'ensemble.

          • Choisissez une année et un mois en haut.
          • Le tableau « Détail par mois » compare le loyer attendu, le montant reçu, la différence et le solde cumulé.
          • Cliquez sur une ligne de mois pour la sélectionner rapidement.
          • Les graphiques et le récapitulatif annuel reprennent les mêmes chiffres sous forme visuelle.
          • La liste « Virements du mois sélectionné » détaille les encaissements du mois choisi.

          Étape 5 Quittance et export

          1. Sélectionnez le bon mois dans le Tableau de bord.
          2. Ouvrez l'onglet Quittance : le texte se remplit automatiquement.
          3. Relisez et corrigez le texte si besoin (comme dans un traitement de texte).
          4. Exportez :
            • PDF — le plus courant pour envoyer ou imprimer ;
            • DOCX — pour modifier dans Word ;
            • HTML — page web autonome.
          5. Pour envoyer par e-mail : Mail (EML + PDF) crée un message prêt à envoyer avec la quittance en pièce jointe.
          Cliquez sur Régénérer si vous avez modifié des virements ou des paramètres et souhaitez recalculer le texte automatique.

          Installation sur un serveur (recommandé)

          Pour éviter de sélectionner un dossier à chaque actualisation, hébergez l'application sur un serveur web avec PHP (présent sur la plupart des hébergeurs mutualisés).

          Étapes

          1. Uploadez le dossier du projet sur votre hébergement (FTP ou gestionnaire de fichiers).
          2. Vérifiez que data/ et templates/ sont inscriptibles par PHP (droits 755 ou 775).
          3. Copiez config.example.php en config.php (optionnel : ajoutez une clé api_key en production).
          4. Ouvrez l'application via son URL (https://votre-domaine.fr/loyer/index.html) — pas en double-cliquant sur le fichier.

          L'application détecte automatiquement api.php et enregistre vos données sur le serveur. Le badge affiche Enregistré sans aucune configuration de dossier.

          Test en local

          Dans le dossier du projet, lancez :

          php -S localhost:8080

          Puis ouvrez http://localhost:8080/index.html dans votre navigateur.

          Sécurité : ne partagez pas l'URL publiquement sans définir une api_key dans config.php. Faites aussi une copie de secours régulière via Paramètres → Données.

          Dépannage

          Les modèles ne s'enregistrent pas

          Vérifiez que nginx et php-fpm tournent, que templates/ est inscriptible par PHP, et que le badge affiche Enregistré. Test : curl http://localhost/api.php?action=status.

          Le badge indique « Serveur indisponible »

          Ouvrez l'app via http:// (pas file://). Vérifiez nginx, php8.4-fpm et les droits sur data/.

          Le fichier JSON est corrompu

          Au démarrage, l'application vous propose de télécharger une copie du fichier endommagé, puis de le recréer vide.

          L'import CSV ne trouve aucun virement

          Vérifiez les « Émetteurs » dans Paramètres : les mots-clés doivent correspondre au libellé affiché par votre banque.

          Les chiffres du tableau de bord semblent incorrects

          Vérifiez la date de début de location, l'historique des loyers et que tous les virements ont bien été saisis.

          Faire une copie de secours

          Paramètres → Données → Copie de secours (export) télécharge votre fichier JSON.
